# Конвенция разбиения (frontend ≤ 700 строк)

Цель: реальная декомпозиция по модулям без изменения поведения и API.

## Принцип

- **Страницы (`pages/**`)**: оставляем файл страницы тонким “компоновщиком”, а логику разносим в соседние подпапки:
  - `pages/<X>/components/**` — UI-блоки/таблички/панели/модалки;
  - `pages/<X>/hooks/**` — хуки состояния/данных/handlers для секций;
  - `pages/<X>/utils/**` — чистые утилиты/форматтеры/валидации.
- **Большие общие компоненты (`components/**`)**: если компонент используется только в одном месте — выносим ближе к потребителю (в `pages/<X>/...`); если используется в нескольких — держим в `components/<domain>/**` и делим на подмодули рядом.
- **API слой (`src/api/queries/**`)**: дробим по доменам (например, `rbac/*`) и оставляем совместимый `re-export` через `src/api/queries/rbac.ts` или `src/api/queries/index.ts`, чтобы минимизировать churn импортов.

## Соглашения

- Не трогаем `frontend/src/api/generated/**`.
- Новые файлы тоже держим ≤ 700 строк; при необходимости — вложенные папки уровня `sections/`.
- Публичные интерфейсы (props/types) — рядом с модулем, где используются; общие типы — в `types.ts` внутри домена.
