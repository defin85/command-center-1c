# Design: UI выбора offline флагов из driver schema

## Контекст
Профиль подключения `ibcmd` на уровне базы хранится как `raw flags` (см. spec `database-connection-profiles`). UI `/databases` даёт пользователю форму редактирования `remote/pid/offline.*`.

Сейчас UI автодобавляет `offline.config` и `offline.data` как строки с пустым `value`, что из-за правил валидации выглядит как “обязательные поля” и блокирует сохранение.

## Цель
Сделать UX предсказуемым:
- никаких дефолтных offline‑строк,
- добавление ключей “по желанию”,
- выбор ключей из актуальной схемы драйвера.

## Источник схемы
Используем существующий публичный (для `IsAuthenticated`) endpoint каталога schema‑driven команд:
- `GET /api/v2/operations/driver-commands/?driver=ibcmd`

Ответ содержит `catalog.driver_schema` (объект). Для offline‑ключей используем:
- `driver_schema.connection.offline` (ожидается объект), ключи объекта = имена offline.* ключей (`config`, `data`, `db_name`, ...).

Примечание: UI не должен полагаться на стабильность структуры beyond этого пути; при отсутствии данных UI остаётся работоспособным (можно добавлять строки вручную).

## UX/поведение
### 1) Нет дефолтных строк
- При открытии модалки `offline_entries` = `[]`, если в профиле ничего не сохранено.
- Сохранённые offline‑пары отображаются как строки (key/value) без добавления новых.

### 2) “Добавить из схемы”
- Над списком offline‑строк размещаем Select/ComboBox “Добавить offline флаг из схемы”.
- Опции строятся из `driver_schema.connection.offline`:
  - label: ключ (например `db_name`) или human label из schema, если доступно,
  - description: текст подсказки (если доступен).
- При выборе опции добавляем строку `{ key: <selected_key>, value: '' }` и фокусируем value.

### 3) Ручное добавление
- Оставляем кнопку “Добавить offline флаг” (добавляет пустую строку key/value).
- Это поддерживает кейсы, когда schema не загрузилась или нужен кастомный ключ.

### 4) Валидация ключей (минимальная)
- Запрещаем ключи, начинающиеся с `-` (например `--db-name`), и подсвечиваем ошибкой:
  - “Ключ вводится без `--` (например `db_name`, а не `--db-name`).”
- Сохраняем запрет на секреты (`db_user/db_pwd/db_password`) как сейчас.

## Тестирование
- Unit‑тесты модалки:
  - при пустом профиле нет дефолтных строк и нет ошибок валидации,
  - добавление строки “из схемы” работает (подменяем ответ каталога),
  - сохранение формирует payload только из заполненных строк.
