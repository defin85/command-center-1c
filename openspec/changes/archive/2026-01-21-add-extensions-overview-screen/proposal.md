# Change: Добавить общий экран расширений (/extensions)

## Why
Сейчас UI показывает расширения только в контексте одной базы (Drawer в `/databases`) и не даёт общего представления о том,
какие расширения установлены/включены на всём парке баз. Операторам нужен обзорный экран, чтобы:
- быстро видеть распределение расширений по базам;
- находить базы без нужного расширения или с неактуальной версией;
- контролировать расхождения (включено/выключено) и устаревшие snapshot'ы.

## What Changes
- Добавить новый экран `/extensions` (таблица “расширение ↔ базы”) с фильтрацией/поиском и drill-down до списка баз.
- Нормализовать (таблично распарсить) список расширений из snapshot'а, чтобы UI мог отображать структурированные поля.
- Добавить API для получения агрегированного представления расширений по доступным пользователю базам.
- Обновить документацию/roadmap.

## Impact
- Затронутые спеки: `extensions-overview` (new)
- Затронутый код (план): Orchestrator API v2 + snapshot pipeline, Frontend новая страница `/extensions`
- Ломающих изменений нет (additive)
