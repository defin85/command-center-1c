# Spec Delta: driver-catalog-kb1ci-import

## ADDED Requirements
### Requirement: Parse-run преобразует DocSet в base catalog v2 для драйвера
Система ДОЛЖНА (SHALL) уметь запускать parse-run для `kb1ci` DocSet и преобразовывать выбранный набор страниц в base catalog v2 для `driver` (`cli`/`ibcmd`).

#### Scenario: Dry-run возвращает отчёт и (опционально) каталог
- **WHEN** администратор запускает parse-run в режиме dry-run
- **THEN** система возвращает `validation_report` (issues[])
- **AND** если ошибок нет, может вернуть собранный каталог в ответе (без публикации)

### Requirement: Строгая schema-валидация результата (fail-closed)
Система ДОЛЖНА (SHALL) валидировать результат парсинга по строгой схеме:
- неожиданные поля запрещены,
- типы полей проверяются,
- обязательные поля должны присутствовать.

Система ДОЛЖНА (SHALL) fail-closed: при ошибках уровня `error` каталог НЕ публикуется/не сохраняется как base.

#### Scenario: Неожиданное поле блокирует publish
- **GIVEN** парсер вернул поле, отсутствующее в schema
- **WHEN** администратор пытается сохранить/опубликовать результат
- **THEN** система возвращает ошибку и указывает путь до проблемного поля
- **AND** base catalog не создаётся

### Requirement: Несколько base catalogs на версию платформы и выбор active
Система ДОЛЖНА (SHALL) поддерживать несколько base catalog версий для одного `driver` в разрезе `platform_version` (например `8.3.23`, `8.3.24`) и позволять выбрать активный набор (active) для каждой версии платформы.

#### Scenario: Администратор выбирает active base для 8.3.23
- **GIVEN** загружено несколько base catalog версий для `(driver=ibcmd, platform_version=8.3.23)`
- **WHEN** администратор выбирает один из них как active
- **THEN** последующие резолвы “active catalog” для `(ibcmd, 8.3.23)` возвращают выбранную версию
