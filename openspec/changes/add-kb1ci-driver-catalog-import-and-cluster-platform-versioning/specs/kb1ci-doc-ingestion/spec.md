# Spec Delta: kb1ci-doc-ingestion

## ADDED Requirements
### Requirement: Синхронизация дерева kb.1ci.com в виде snapshot
Система ДОЛЖНА (SHALL) уметь синхронизировать дерево документации `kb.1ci.com` (XWiki) через REST API и сохранять результат как snapshot, включающий:
- стабильные идентификаторы документов (`doc_id`),
- связи parent/children,
- метаданные (`title`, `url`, `syntax`, `language`, `translations`, `version`, `modified`),
- raw контент страницы (минимум `content` в `xwiki/2.1`) для страниц, включённых в выбранные DocSets.

#### Scenario: Snapshot создаётся и доступен для UI
- **WHEN** staff/admin запускает sync KB
- **THEN** система создаёт новый snapshot и возвращает его id/статус
- **AND** UI может запросить дерево разделов из последнего snapshot

### Requirement: UI показывает полный список разделов и позволяет выбрать DocSet
Система ДОЛЖНА (SHALL) предоставить UI, где администратор видит полный список разделов (дерево) KB и может создать DocSet, выбрав:
- корневые страницы/разделы,
- режим включения (только корни / всё поддерево),
- target driver (`cli`/`ibcmd`) и target `platform_version` (например `8.3.23`).

#### Scenario: Администратор создаёт DocSet без авто-маппинга
- **WHEN** администратор создаёт DocSet
- **THEN** система сохраняет выбранные корни/режим включения
- **AND** система НЕ пытается автоматически определять “какой раздел для какой схемы” — это ручная настройка

### Requirement: i18n-ready метаданные без breaking по умолчанию
Система ДОЛЖНА (SHALL) сохранять информацию о доступных переводах документа (`translations`) и языке контента, сохраняя при этом стабильные идентификаторы (`doc_id`) независимо от языка.

#### Scenario: Страница имеет переводы
- **GIVEN** страница KB имеет несколько языковых версий
- **WHEN** система сохраняет её в snapshot
- **THEN** в метаданных указаны доступные language codes и default language
