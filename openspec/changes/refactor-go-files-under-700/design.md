# Дизайн: декомпозиция Go файлов внутри пакета

## Принцип
В Go “модульность” достигается разносом по файлам внутри **того же пакета**:
- приватные функции остаются доступными в пакете;
- публичные символы не обязаны менять имена;
- уменьшается размер единицы чтения.

## Рекомендованный шаблон
- `*_types.go` — типы/константы/ошибки
- `*_handlers.go` — обработчики/входные точки
- `*_builder.go` — сборка/конструирование (argv/config)
- `*_storage.go` — persistence/IO
- `*_test_*.go` — тесты по сценариям

## Риски
- Неочевидные зависимости между файлами (порядок инициализации `init()`).
- Тесты, завязанные на общий `TestMain`/общие фикстуры, могут потребовать выделения общего `test_helpers.go`.
