# Change: Рефакторинг Go исходников до целевого размера (700 строк)

## Why
В Go сервисах есть крупные файлы (700–1200+ строк) и большие тесты. Это осложняет сопровождение и снижает удобство работы с кодом (включая LLM/агентов).

## What Changes
Рефакторим Go код (включая тесты) так, чтобы:
- разнести большие файлы на несколько `*.go` внутри того же пакета по ответственности;
- привести размер каждого файла к целевым ~700 строк;
- не менять поведение сервисов и публичные интерфейсы пакетов.

Крупнейшие файлы (примерный срез):
- `go-services/worker/internal/drivers/ibcmdops/driver.go` (~1231)
- `go-services/worker/internal/saga/orchestrator.go` (~926)
- `go-services/worker/internal/queue/stream_consumer.go` (~787)
- крупные тесты saga/resource manager/integration (~700–860)

## Non-Goals
- Не меняем бизнес-логику и протоколы.
- Не делаем жёсткий CI gate на размер файлов.

## Impact
- Разнос больших файлов на пакетные подмодули.
- Точечные правки импортов/пакетных приватных функций.
