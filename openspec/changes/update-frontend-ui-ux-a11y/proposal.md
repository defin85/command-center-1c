# Change: UI/UX/a11y аудит и приведение фронта к Web Interface Guidelines

## Why
Часть UI элементов сейчас не соответствует Web Interface Guidelines (в т.ч. базовым a11y правилам), что ухудшает:
- доступность (клавиатурная навигация, скринридеры, корректные labels);
- качество UX (некорректные подсказки, неоднозначные интерактивные зоны);
- консистентность интерфейса (типографика: `…` vs `...`).

## Scope
Аудит охватывает:
- `frontend/src/pages/**` (основные страницы);
- ключевые shared компоненты, влияющие на страницы: `frontend/src/components/**`, `frontend/src/hooks/**`, `frontend/src/stores/**`.

## What Changes
- Исправить a11y/UX нарушения из Findings, не меняя доменную логику.
- Добавить минимальные тесты/проверки на ключевые регрессии (icon-only aria-label, keyboard handlers, labels).

## Non-Goals
- Не меняем API/контракты и бизнес-логику.
- Не проводим редизайн; только локальные UI/a11y правки.

## Impact
- Затронуты UI страницы и shared компоненты фронта.
- Риски: минимальные (правки в разметке/атрибутах), но нужно проверить keyboard flow и regression в тултипах/поповерах.
