FEATURES
================================================================================

AUTOMATIC REPLAY
  - Scheduled every 60 seconds via Celery Beat
  - Batch processing (configurable, default: 100 events)
  - Reconnects to Redis on each execution
  - Publishes to Redis Streams (XADD)

RETRY LOGIC
  - Exponential backoff for failures
  - Configurable max retries (default: 5)
  - Increments retry_count on each failure
  - Marks event as FAILED when max retries exceeded

AUTOMATIC CLEANUP
  - Scheduled daily at 04:00 UTC
  - Removes replayed events older than configured days (default: 7)
  - Prevents unbounded database table growth
  - Logs number of deleted records

ERROR HANDLING
  - Catches Redis connection errors
  - Logs warnings and errors comprehensively
  - Per-event error tracking (last_error field)
  - Continues batch processing on individual failures
  - Safe for crashes (idempotent, atomic updates)

MONITORING
  - Event status tracking (PENDING, REPLAYED, FAILED)
  - Correlation IDs for tracing
  - Detailed error messages for debugging
  - Django admin integration
