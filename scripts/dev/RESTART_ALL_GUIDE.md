# restart-all.sh - Умный перезапуск с автоматической пересборкой

## Назначение

Скрипт `restart-all.sh` выполняет интеллектуальный перезапуск всех сервисов CommandCenter1C с автоматическим определением изменений в Go коде и выборочной пересборкой только измененных сервисов.

## Основные возможности

- **Smart Rebuild**: Автоматическое определение изменений в `.go` файлах
- **Selective Rebuild**: Пересборка только измененных сервисов (экономия времени)
- **shared/ Detection**: Автоматическая пересборка всех сервисов при изменении `go-services/shared/`
- **Single Service Mode**: Перезапуск одного конкретного сервиса
- **Force Rebuild**: Принудительная пересборка всех Go сервисов
- **No Rebuild Mode**: Только перезапуск без проверки/пересборки
- **Parallel Build**: Параллельная сборка для ускорения

## Использование

### Базовый сценарий (Smart Restart)

```bash
./scripts/dev/restart-all.sh
```

**Что происходит:**
1. Проверка каждого Go сервиса на изменения (сравнение timestamp исходников и бинарников)
2. Проверка `go-services/shared/` на изменения
3. Пересборка только измененных сервисов (или пропуск если все актуально)
4. Остановка всех сервисов через `stop-all.sh`
5. Запуск всех сервисов через `start-all.sh`
6. Итоговая сводка

### Принудительная пересборка

```bash
./scripts/dev/restart-all.sh --force-rebuild
```

Пересобрать все Go сервисы независимо от изменений.

### Только перезапуск (без rebuild)

```bash
./scripts/dev/restart-all.sh --no-rebuild
```

Пропустить проверку изменений и пересборку, только перезапустить сервисы.

### Перезапуск одного сервиса

```bash
./scripts/dev/restart-all.sh --service=api-gateway
```

Перезапустить только API Gateway (с проверкой изменений и пересборкой при необходимости).

**Доступные сервисы:**
- `orchestrator`
- `celery-worker`
- `celery-beat`
- `api-gateway`
- `worker`
- `ras-grpc-gw`
- `cluster-service`
- `batch-service`
- `frontend`

### Параллельная пересборка

```bash
./scripts/dev/restart-all.sh --parallel-build
```

Использовать параллельную сборку Go сервисов (быстрее при пересборке нескольких сервисов).

### Детальный вывод (для отладки)

```bash
./scripts/dev/restart-all.sh --verbose
```

Показать детальную информацию о проверке изменений и процессе пересборки.

### Комбинация флагов

```bash
# Принудительная параллельная пересборка с детальным выводом
./scripts/dev/restart-all.sh --force-rebuild --parallel-build --verbose

# Перезапуск одного сервиса без rebuild
./scripts/dev/restart-all.sh --service=worker --no-rebuild
```

## Алгоритм определения изменений

### Для каждого Go сервиса:

1. **Проверка бинарника**: Если `bin/cc1c-<service>.exe` не существует → `REBUILD_NEEDED`
2. **Поиск самого нового `.go` файла**: `find <service-dir> -name "*.go" -type f`
3. **Сравнение timestamps**: Если исходник новее бинарника → `REBUILD_NEEDED`
4. **Проверка shared/**: Если `go-services/shared/*.go` новее бинарника → `REBUILD_NEEDED`
5. Иначе → `UP_TO_DATE`

### Проверка shared/ модулей:

Если хотя бы один файл в `go-services/shared/*.go` новее **самого старого** бинарника → пересобрать **ВСЕ** Go сервисы.

## Примеры сценариев

### Сценарий 1: Изменен один сервис

```bash
# Изменил go-services/api-gateway/cmd/main.go
./scripts/dev/restart-all.sh
```

**Результат:**
- Пересоберется только `api-gateway`
- Остальные сервисы останутся без изменений
- Все сервисы перезапустятся

### Сценарий 2: Изменен shared/ модуль

```bash
# Изменил go-services/shared/config/config.go
./scripts/dev/restart-all.sh
```

**Результат:**
- Пересоберутся **ВСЕ 4** Go сервиса (`api-gateway`, `worker`, `cluster-service`, `batch-service`)
- Все сервисы перезапустятся

### Сценарий 3: Бинарники актуальны

```bash
# Не менял код, просто хочу перезапустить
./scripts/dev/restart-all.sh
```

**Результат:**
- Пересборка пропущена (все актуально)
- Все сервисы перезапустятся

### Сценарий 4: Быстрый restart без rebuild

```bash
# Изменил Python/JS код, Go не трогал
./scripts/dev/restart-all.sh --no-rebuild
```

**Результат:**
- Пересборка пропущена (флаг `--no-rebuild`)
- Все сервисы перезапустятся

## Сравнение с restart.sh

| Функция | `restart.sh` | `restart-all.sh` |
|---------|-------------|------------------|
| Перезапуск одного сервиса | ✅ | ✅ (`--service=`) |
| Перезапуск всех сервисов | ❌ | ✅ |
| Автоматическая пересборка Go | ❌ | ✅ |
| Определение изменений | ❌ | ✅ |
| Проверка shared/ | ❌ | ✅ |
| Выборочная пересборка | ❌ | ✅ |

**Рекомендация:**
- `restart.sh` - для быстрого перезапуска одного сервиса без rebuild
- `restart-all.sh` - для полного перезапуска с умной пересборкой

## Устранение неполадок

### Скрипт не находит бинарники

```bash
# Собрать все бинарники
./scripts/build.sh

# Или использовать --force-rebuild
./scripts/dev/restart-all.sh --force-rebuild
```

### Ошибка "конфликтующие флаги"

```
Ошибка: Флаги --force-rebuild и --no-rebuild несовместимы
```

**Причина:** Нельзя одновременно использовать `--force-rebuild` и `--no-rebuild`.

**Решение:** Выберите один из флагов.

### Скрипт всегда пересобирает все сервисы

**Причина:** Возможно изменен `go-services/shared/`.

**Диагностика:**
```bash
./scripts/dev/restart-all.sh --verbose
```

Посмотрите на вывод секции "Проверка shared/ модулей".

### Медленная пересборка

**Решение:** Используйте параллельную сборку:
```bash
./scripts/dev/restart-all.sh --parallel-build
```

## Интеграция с другими скриптами

Скрипт делегирует работу существующим скриптам:

```
restart-all.sh
├── scripts/build.sh          (пересборка Go)
├── scripts/dev/stop-all.sh   (остановка сервисов)
├── scripts/dev/start-all.sh  (запуск сервисов)
└── scripts/dev/restart.sh    (перезапуск одного, если --service=)
```

Это обеспечивает консистентность и упрощает поддержку.
