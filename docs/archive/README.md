# Архив документации CommandCenter1C

Этот каталог содержит архивированную документацию, которая была перенесена из активной части проекта. Документы сохранены для исторической справки и могут быть полезны для понимания эволюции проекта.

---

## 📁 Структура архива

### `research/ras_rac/` (6 файлов, 86KB)

**Что:** Исследовательская работа по RAS/RAC администрированию 1С

**Файлы:**
- `1C_RAS_vs_RAC.md` - Сравнение RAS и RAC подходов
- `1C_RAC_COMMANDS.md` - Справочник команд RAC CLI
- `1C_RAC_SECURITY.md` - Ограничения безопасности RAC
- `1C_RAS_API_OPTIONS.md` - Опции RAS API
- `1C_RAS_ALL_API_OPTIONS.md` - Полный список опций RAS API
- `1C_RAS_GRPC_SOLUTION.md` - gRPC решение через ras-grpc-gw

**Почему архивировано:**
Ценная исследовательская работа, но детализация избыточна для текущих нужд. Для ежедневного использования достаточно краткого руководства.

**Актуальная замена:**
- `docs/1C_ADMINISTRATION_GUIDE.md` (будущий консолидированный документ)

---

### `django_cluster_sync/` (4 файла, 45KB)

**Что:** Документация по интеграции cluster-service с Django Orchestrator

**Файлы:**
- `DJANGO_CLUSTER_SYNC.md` - Общая концепция синхронизации
- `CLUSTER_SERVICE_IMPLEMENTATION.md` - Детали реализации gRPC клиента
- `CLUSTER_SYNC_IMPLEMENTATION.md` - Синхронизация моделей Django
- `CLUSTER_SYNC_SUMMARY.md` - Краткое резюме

**Почему архивировано:**
Актуальная функциональность, но информация дублируется в 4 местах с разными деталями реализации.

**Актуальная замена:**
- `docs/DJANGO_CLUSTER_INTEGRATION.md` (будущий консолидированный документ)
- `go-services/cluster-service/README.md` - документация cluster-service

---

### `odata/` (3 файла, 53KB)

**Что:** Документация по интеграции OData для работы с 1С

**Файлы:**
- `TZ_ODATA_AUTOMATION.md` - Техническое задание и настройка OData в 1С
- `README.md` - Общее описание OData integration
- `IMPLEMENTATION.md` - Детали реализации Python клиента

**Почему архивировано:**
Актуальная функциональность, требует консолидации в единый практический гайд.

**Актуальная замена:**
- `docs/ODATA_INTEGRATION.md` (будущий консолидированный документ)
